# 자동추천 탭 확장 계획

## 현재 확인한 구조
- 자동추천 탭 경로는 `/menu-4`이고, 진입 컴포넌트는 `src/app/menu-4/page.tsx`입니다.
- 실제 화면 로직은 `src/components/pages/crawl-orders-page.tsx`의 `CrawlOrdersPage`에 모여 있습니다.
- 기존 "자동추천" 목록은 `crawl_orders`에서 `user_id`와 `crawl_order_status = 0` 조건으로 조회하고, 선택 시 `?id=<crawl_order_id>` 상세 화면으로 이동합니다.
- 입금 내역 테이블은 `bank_account_deposit`, 계좌 테이블은 `bank_account`이며, `bank_account_deposit.bank_account_id -> bank_account.id` 관계가 타입과 DB 문서에 있습니다.
- 주문의 입금 완료 여부는 `orders.is_processed`로 판단할 수 있고, 미완료 주문은 `is_processed = false`로 조회할 수 있습니다.

## 작업 목표
1. 자동추천 탭을 좌우 슬라이드 가능한 2페이지 구조로 바꿉니다.
2. 첫 페이지는 기존 크롤링 주문 추천 목록을 유지하되 상단 문구를 아래처럼 바꿉니다.
   - 제목: `주문 내역 자동 추천`
   - 설명: `구매장부에 등록되지 않은 구매 주문 건을 표시합니다`
3. 오른쪽 스와이프 또는 오른쪽 화살표로 두 번째 페이지로 이동할 수 있게 합니다.
4. 두 번째 페이지는 아래 상단 문구와 함께 입금 내역 자동 추천 목록을 보여줍니다.
   - 제목: `입금 내역 자동 추천`
   - 설명: `입금완료 처리되지 않은 미완료 주문건을 표시합니다`
5. 입금 내역 자동 추천 페이지는 `bank_account_deposit` 목록을 `bank_account`와 조인해 보여주고, 화면에는 `bank_account.bank_account_name`만 노출합니다.
6. 입금 건을 클릭하면 드롭다운/펼침 영역으로 추천 미완료 주문건을 보여줍니다.

## 구현 계획
1. Next.js 문서 확인
   - `node_modules/next/dist/docs/`에서 현재 프로젝트의 App Router/Client Component 관련 문서를 먼저 확인합니다.
   - 검증: 문서 확인 후 기존 `use client`, `useSearchParams`, `Suspense` 구조와 충돌하지 않는 방식으로만 수정합니다.

2. 기존 자동추천 화면 구조 분리
   - `CrawlOrdersPage` 안의 기존 목록 UI를 "주문 내역 자동 추천" 페이지로 유지합니다.
   - 단일 파일 내에서 필요한 만큼만 작은 렌더링 단위로 정리하되, 새 파일 생성은 중복/복잡도가 커질 때만 검토합니다.
   - 검증: 기존 크롤링 주문 목록, 상세 진입, 삭제 처리, 크롤링 실행 버튼 동작이 유지됩니다.

3. 좌우 슬라이드 페이지 UI 추가
   - 모바일/앱 WebView에서는 터치 스와이프가 동작하도록 하고, 데스크톱/모바일 공통으로 좌우 화살표 버튼을 제공합니다.
   - 두 페이지가 같은 폭으로 나란히 배치되고 현재 페이지 인덱스에 따라 `transform`으로 이동하는 방식이 가장 단순해 보입니다.
   - 검증: 작은 화면에서도 제목, 설명, 계정 메뉴, 화살표, 주요 목록이 겹치지 않습니다.

4. 입금 내역 조회 추가
   - 로그인 사용자 확인 후 `bank_account_deposit`을 `bank_account`와 조인해 조회합니다.
   - 표시 필드는 최소한 입금일, 입금시각, 상대방, 금액, `bank_account_name`으로 구성합니다.
   - 정렬은 질문 답변에 따라 확정합니다.
   - 검증: `bank_password`, `resident_number`, `bank_account_number` 같은 민감 정보는 조회/표시하지 않습니다.

5. 미완료 주문 추천 로직 추가
   - 미완료 주문은 `orders.is_processed = false`로 조회합니다.
   - 입금 건 클릭 시 아래 우선순위로 추천 주문을 계산합니다.
     1. `bank_account_deposit.counterparty`와 `orders.title`이 정확히 일치하는 미완료 주문
     2. 1번 결과가 없고 `counterparty` 앞 네 글자가 숫자이면, 그 값을 `MMDD`로 보고 `orders.purchase_date`의 월/일과 일치하는 미완료 주문
   - 검증: 1번 매칭 결과가 있으면 2번 날짜 매칭은 보여주지 않습니다.

6. 입금 내역 추천 UI 구성
   - 기존 자동추천 탭의 카드/테이블 톤을 재사용합니다.
   - 모바일은 카드 목록, 데스크톱은 테이블 형태로 맞추는 방향이 기존 디자인과 가장 일관됩니다.
   - 입금 건 클릭 시 같은 행/카드 아래에 추천 미완료 주문 목록을 펼칩니다.
   - 검증: 추천 주문이 없는 경우에도 빈 상태 문구가 표시됩니다.

7. 문서 업데이트
   - 코드 수정 후 `docs/project_overview.md`에 자동추천 탭이 2페이지 구조가 되었고, 입금 내역 자동 추천이 `bank_account_deposit`/`bank_account`/`orders`를 사용하는 점을 반영합니다.
   - DB 스키마 변경은 계획에 없으므로 `docs/guide_db.md`는 수정 대상이 아닙니다.
   - 검증: 문서가 구현 결과와 맞습니다.

8. 최종 검증
   - `npm run lint`를 실행합니다.
   - 가능하면 로컬 개발 서버에서 `/menu-4`를 열어 데스크톱/모바일 폭을 확인합니다.
   - 검증: 기존 주문 내역 자동 추천과 새 입금 내역 자동 추천이 모두 표시되고, 좌우 이동과 펼침 동작이 동작합니다.

## 확인 질문
1. 입금 내역 자동 추천에서 `bank_account_deposit`은 어떤 범위를 보여주면 될까요?
   - 예: 전체 입금 내역, 최근 N일, 아직 어떤 주문에도 반영되지 않은 것으로 추정되는 입금 내역만 등
   - 답변: bank_account_deposit테이블에 bank_account_deposit_status컬럼을 만들었음 (미완료 0, 매핑완료 1, 삭제 99) status가 0인 건들 (미완료)만 보여주면 됨

2. 입금 건 정렬 기준은 무엇이 좋을까요?
   - 예: `date` 내림차순 + `time` 내림차순, 또는 최신 등록순
   - 답변: `date` 오름차순 + `time` 오름차순

3. `counterparty`와 `orders.title` 비교는 공백/대소문자/특수문자를 그대로 둔 "완전 일치"가 맞나요?
   - 현재 요청 문구는 "일치"라서 기본 계획은 양쪽을 `trim()`한 정확 일치입니다.
   - 답변: trim()한 일치로 해주는데 비슷한게 있으면 일치율 **%로 같이 띄워주면 더 좋음 정렬은 일치율이 높은 순서대로 최대 3개


4. `counterparty` 앞 네 글자를 `MMDD`로 간주할 때 연도는 어떤 기준으로 잡을까요?
   - 제안: `orders.purchase_date` 전체 날짜가 아니라 월/일만 비교합니다. 예를 들어 `0525`이면 모든 연도의 5월 25일 미완료 주문이 후보가 됩니다.
   - 답변: 연도는 무시

5. 입금 건을 클릭해 추천 주문을 보여준 뒤, 주문을 선택했을 때 어떤 동작이 필요할까요?
   - 예: 주문 상세로 이동만, 기존 구매장부 완료처리 패널처럼 입금일/금액/메모를 채워 완료처리까지 가능하게 하기, 이번 단계에서는 추천 목록 표시만
   - 답변: 입금일: 입금날짜, 금액: amount, 메모: counterpary 로 해서 완료주문 처리, bank_account_deposit.bank_account_deposit_status을 1로 처리 

6. 입금 내역 자동 추천의 집계 카드도 기존 주문 추천처럼 3개를 둘까요?
   - 예: 입금 내역 수, 자동 매칭 수, 확인 필요 수
   - 답변: 카드는 없애줘, 대신에 내 계정이랑 매핑된 bank_account 행만 보여주면 돼 내용은 bank_account_name, bank, bank_account_number 이렇게 
7. 두 페이지 전환 상태를 URL에 남길까요?
   - 예: `/menu-4?tab=deposit`처럼 새로고침/뒤로가기 후에도 입금 내역 자동 추천 페이지 유지
   - 답변: 아니오
